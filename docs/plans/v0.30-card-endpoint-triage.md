# oncbrain study-card redesign — implementation plan (scoped v2)

> Rewritten after adversarial + codex review of v1. v1 was ~2-3× over-scoped: it
> reinvented shipping features, moved the headline number out of the validated
> path, rendered ungated LLM booleans as clinical UI, and risked the publish
> boundary. This version keeps only what survives all three reviews.

## Goal (reframed)

**Sharper triage, not a re-architecture.** Two changes:
1. **Kill the redundant interpretive prose** (the TL;DR / verdict-rationale / significance / monday_clinic overlap) — the one clearly-observed win.
2. **One grounded primary-endpoint line** in the head, only when the source states a primary endpoint — so a reader sees *which* endpoint (survival vs surrogate vs local) alongside the number, without guessing.

Everything else the card already does (figures column, lightbox, CONSORT, trials-to-watch, IMRD fold, monday_clinic dedup) stays as-is. Product register unchanged: 90-second forwardable triage artifact, DESIGN.md "brevity beats completeness."

## Explicit non-goals (cut from v1, with why)

- **Negative-result banner / NS-tag / muted number** — the `challenges-soc` / `methodologically-limited` verdict plus a `⚠️` critique bullet already carry this (VOICE.md ships the canonical "OS HR 0.87, not significant" bullet). A new visual grammar misfires on the ~21% non-inferiority/de-escalation cards where "not significant" is the *intended* result.
- **`discussion` field** — duplicates `significance`; a second long-form prose block violates VOICE.md bullet-density and deepens perspective lock-in on a forwardable artifact.
- **`represents` / `excludes` fields** — reuse `verdict.audience` (StudyCard.astro:310) and the `monday_clinic` boundary clause.
- **"Paper-replacement / carry every datum"** — inverts DESIGN.md; the fold already drops no bullet (the `notes` bucket exists for exactly this, StudyCard.astro:209).
- **Figures / lightbox / CONSORT / trials / desktop-column work** — already shipped (figure column StudyCard.astro:856; lightbox `.study-zoom` :818 + Base.astro:482; `+N more` :466; CONSORT :580; trials-to-watch :610). Do not rebuild.

## Corpus reality (measured, 100 studies / 32 digests)

26% have an NCT · 25% randomized · 16% reviews · 33% observational · 12% single-arm · 50% printed p · 41% printed CI · 88% already have `significance` · 92% already have `related_trials`. → the endpoint chip is meaningful on the ~25% RCT slice and must **abstain** elsewhere, not manufacture an endpoint.

## Phase 0 — prerequisites (BEFORE any new field)

1. **Recursive publish-boundary test.** Extend `test/publish-boundary.test.ts` (currently checks only `artifact.papers`, :52) to assert the forbidden keys (`figure_structured_md`, `figure_ocr_md`, `fulltext_excerpt_md`, PDF page paths) never appear **anywhere** in the artifact JSON, the `/api/v1/*` JSON, or rendered HTML. Needed because `sanitizeArtifactForApi` spreads the whole artifact (`api-output.ts:78`) and per-study API carries the whole `study` (`:107`, `:133`), so any new field auto-publishes.
2. **Measure Phase-2 output size** on the 3 densest committed cards against the `maxTokens: 4096` cap (`llm-pipeline.ts:1163`, single retry `:746`, repair reuses the cap `:2720`). More output silently drops studies (truncated JSON → parse fail). If headroom < one small field, that gates Phase 2.

## Phase 1 — Monday-clinic ordering (no schema change, ships alone) — DONE

Empirical finding (measured across all 100 committed cards) killed the originally
planned **lexical** dedup:
- `significance`≈`tldr` token-overlap ≥0.6 fires on **19** cards, but inspection
  (OPERA 0.76, SWOG 0.75, PIVOTALboost 0.63) shows those are ADDITIVE elaboration that
  reuses the numbers/drug names, not restatement — a render drop would lose content.
- `monday_clinic`≈`significance` lexical overlap ≥0.6 is **0** cards: the redundancy a
  reader perceives is SEMANTIC, which a render-layer text compare can't safely catch.
  (The Phase-2 prompt already has an abstain-if-restates rule; tightening it is a
  prompt/rebuild follow-up, not a render fix.)

**Shipped instead** (pure render, drops nothing, whole corpus immediately): the folded
`Monday clinic` block now renders AFTER the evidence sections (Methods → Results →
Critique) instead of leading the fold, so the bottom-line conclusion follows its data
instead of preceding it (`StudyCard.astro`; `astro check` clean, publish-boundary
green). The safe, evidence-backed answer to the original "is that a conclusion that
should follow the results?" observation.

**Deferred** (needs a rebuild, not render): strengthen the Phase-2 `monday_clinic`
abstain-if-restates-`significance` rule to cut the SEMANTIC overlap at generation time.

Phase-0 hardening follow-up (from confirmation review): the boundary test is a
*denylist* over still-spread study objects; a study-field *allowlist* in
`sanitizeArtifactForApi` would make it structural. Deferred (higher API-regression risk).

## Phase 2 — grounded primary-endpoint chip (ONE new field)

**Schema:** add one optional field to *both* `DigestStudy` definitions (`llm-pipeline.ts` + `digest-data.ts` — lockstep; `astro check` catches renderer drift, but also add it to the parse constructor `parseStudyAgentResponse` :1429, since emit-side drift is silent):

```ts
primary_endpoint?: {
  name: string;          // "Metastasis-free survival" — the source's STATED primary
  klass: 'overall-survival' | 'surrogate' | 'local-control' | 'safety';
  stat_text: string;     // verbatim as printed: "HR 0.53 (95% CI 0.39–0.71), p<0.001"
} | null;
```

**Prompt** (`digest-v5-study-agent.txt`): emit `primary_endpoint` **only when the source states a primary endpoint** (registered RCTs). `stat_text` is verbatim (effect size + p/CI exactly as printed). Abstain → `null` for reviews / single-arm / observational / abstract-only. No new numbers, ever.

**Build-time derivation (deterministic, NOT the LLM):**
- Validate the number(s) in `stat_text` with the existing source-number check (reuse the `firstUnverifiedCellValue` path that guards `details[].table` / caption cells, `llm-pipeline.ts:1272`/`:2387`). If unverified → set `primary_endpoint = null` (never publish an unvalidated headline number). This keeps the headline number **on** the validated path, not off it.
- `significant` is **derived** at build by regex on `stat_text` (printed `p<0.05` OR a CI excluding 1.0) → `true`/`false`; `null` when no p/CI printed. Never an LLM boolean.

**Render** (`StudyCard.astro`, head): a compact endpoint line — `name` + `klass` chip + `stat_text`. No banner, no muting. When `significant===false` on a superiority trial, the existing `⚠️` Critique bullet carries the "primary not significant" read (reinforce the prompt to emit that bullet; it already fits VOICE.md:319). Fallback: `primary_endpoint == null` → today's TL;DR head unchanged (the 100-card corpus renders exactly as now).

## Phase 3 — (optional, later) significance into the critique bullet

No new field. Reinforce the Phase-2 prompt to reliably emit a `⚠️` critique bullet when a **superiority** trial's primary isn't significant, gated by trial-design awareness in the prompt so non-inferiority successes are never flagged. Reuses the existing bullet path + VOICE pattern.

## Grounding rules (hard)

- `stat_text` verbatim from source; the number is validated against source tokens or the field is dropped.
- `significant` derived deterministically at build; never trusted from the model.
- Existing "no invented NCT/citation" assert stays.
- Abstain (`null`) beats guessing on `primary_endpoint`, `klass`, and the primary pick.

## Consumers / drift

- Two `DigestStudy` defs updated in lockstep + the parse constructor (3 edit sites, not 2).
- `feed.ts`, `obsidian-export.ts`, `share-image.ts`, search index: ignore the new field (no change needed); confirm none crash on its absence.
- `primary_endpoint` is publishable summary (verbatim, validated) → auto-publishes via the API spread, which is fine; the Phase-0 recursive boundary test confirms no *local-only* field leaks alongside it.

## Testing

- Phase-0 recursive publish-boundary (artifact + API + HTML).
- Endpoint number validation: unverifiable `stat_text` → `primary_endpoint` null.
- Significance derivation: `p=0.088` → false, `p<0.001` → true, no p/CI → null; a CI including 1.0 → false.
- Dedup: `monday_clinic ≈ tldr` → single render.
- Phase-0 token-size measurement recorded.
- `npx astro check` clean; existing 1000+ vitest pass.

## Residual risks

- **Primary-endpoint pick correctness** — grounded in the source's stated primary; abstain when unclear. Lower blast radius than v1 because there's no banner and the number is validated.
- **Token headroom** — measured in Phase 0; only one small field added.
- **Reviewer-flagged blockers (v1)** — all four P1s (ungated booleans, number off validated path, auto-publish, IP boundary) are resolved by: deterministic significance, validator reuse, recursive boundary test, and adding no figure/structured-extract field.

## Review provenance

Scope set by cross-model review of v1 (2 adversarial red-teams + codex), which unanimously returned "rework/scope-down." This v2 keeps only the pieces all three endorsed: prose dedup + a validator-gated endpoint chip + significance via the existing critique bullet.
